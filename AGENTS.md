# Agent Instructions

- Treat this repository as a public project. Do not commit secrets, private planning notes, implementation logs, PRDs, TRDs, or local environment files.
- Make changes on a feature branch and raise a pull request for review instead of pushing directly to `main`.
- When asked to push a new release, bump the VibeStack version number before opening the release pull request. Update the root package and workspace package versions together.
- Release pull requests must include the intended version, a concise change summary, and the validation commands that were run.
- After a release PR is merged, create or move the release tag only when release publication is explicitly confirmed.
